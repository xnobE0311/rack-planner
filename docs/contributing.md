# Contributing to Rack Planner

We appreciate your interest in contributing to Rack Planner! This document outlines the guidelines to help you make effective contributions.

## Contribution Guidelines
1. **Fork the Repository**: Start by forking the repository to your own account and clone it to your local machine.
2. **Create a Branch**: Make sure to create a new branch for each feature or bug fix you're working on. Use a descriptive name for your branch (e.g., `feature/add-new-component`, `bugfix/fix-issue-1`).
3. **Write Clear Commit Messages**: Keep your commits focused and write meaningful commit messages.

## Code Standards
- Follow the coding style used in the project. Make sure your code is well-organized and readable.
- Use meaningful variable and function names.
- Adhere to the DRY (Don't Repeat Yourself) principle.

## Pull Request Process
1. **Open a Pull Request**: Once you have completed your changes, push your branch to your forked repository and open a pull request (PR) against the `main` branch of the original repository.
2. **Provide a Description**: Include a clear description of what changes you made and why. Reference any issues your PR addresses.
3. **Make Revisions**: Be open to feedback and make changes as necessary. Address any comments made by the reviewers promptly.

## Development Workflow
- Utilize issues to track new features, bugs, or enhancements. Before beginning work on a new feature, please check if there is an open issue.
- Regularly sync your fork with the main repository to avoid merge conflicts.
- Consider running tests before submitting a PR to ensure your changes don't break existing functionality.

Thank you for contributing to Rack Planner! We look forward to your contributions!
